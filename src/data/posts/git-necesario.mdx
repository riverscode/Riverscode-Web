## Básicos

### Configuración especifica de usuario

```bash:terminal
git config --global user.name "Rivers Code"
git config --global user.email "riverscode@gmail.com"
```

### Configuración global

```bash:terminal
git config --global core.editor "code --wait"
git config --global color.ui true
git config --global init.defaultBranch main
```
### Ver la configuración

```bash:terminal
git config --list
```

### Crear alias para comandos

```bash:terminal
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
```

### Inicializar un repositorio

```bash:terminal
git init
```

## Rebase

Git rebase se usa para aplicar los cambios de una rama a otra. Por ejemplo, si estás trabajando en una rama y quieres agregar los cambios de otra rama a la tuya, puedes usar git rebase para agregar los cambios de la otra rama a la tuya.

### Cómo rebase

```bash:terminal
git rebase nombre-rama
```

## Revertir cambios

### Revertir un commit

```bash:terminal
git revert id-commit
```

### Revertir un commit con un mensaje

```bash:terminal
git revert -m 1 id-commit
```

### Devolve todo al principio del rebase

```bash:terminal
git rebase --abort
```

## Trabajar con ramas

### Crear una rama

```bash:terminal
git branch nombre-rama
```

### Cambiar de rama

```bash:terminal
git checkout nombre-rama
```

### Crear y cambiar en nueva rama

```bash:terminal
git checkout -b nombre-rama
```

### Eliminar una rama

```bash:terminal
git branch -d nombre-rama
```

### Eliminar una rama forzada


```bash:terminal
git branch -D nombre-rama
```

### Ver las ramas

```bash:terminal
git branch
```

### Ver las ramas y sus commits

```bash:terminal
git branch -v
```

## Stash

El comando git stash almacena temporalmente (o guarda en un stash) los cambios que hayas efectuado en el código en el que estás trabajando para que puedas trabajar en otra cosa y, más tarde, regresar y volver a aplicar los cambios más tarde. Guardar los cambios en stashes resulta práctico si tienes que cambiar rápidamente de contexto y ponerte con otra cosa, pero estás en medio de un cambio en el código y no lo tienes todo listo para confirmar los cambios.

### Cómo guardar el trabajo en un stash

```bash:terminal
git stash
```

### Cómo ver los stashes

```bash:terminal
git stash list
```

### Cómo recuperar los cambios de un stash

```bash:terminal
git stash apply
```

### Cómo recuperar los cambios de un stash y eliminarlo

```bash:terminal
git stash pop
```

## tags

Los tags son etiquetas que se pueden aplicar a un commit específico. Los tags se utilizan para marcar puntos específicos en la historia de un proyecto, como versiones de lanzamiento, por ejemplo.

### Cómo crear un tag

```bash:terminal
git tag -a v1.0.0 -m "Versión 1.0.0"
```

### Cómo ver los tags

```bash:terminal
git tag
```

### Cómo ver los tags y sus commits

```bash:terminal
git tag -n
```

### Enviar tags a github

```bash:terminal
git push origin --tags
```
### Estado del repositorio

```bash:terminal
git status -s
```

### Listado de commits

  ```bash:terminal
  git log --graph --decorate --oneline --all
  ```
